## How this site is organised

The material is split into two big buckets, which is the whole point of DSA:

- **[Data Structures](data-structures/index.md)** — the *containers* that hold data: arrays, linked lists, stacks, queues, hash tables, trees, and graphs.
- **[Algorithms](algorithms/index.md)** — the *procedures* that act on data: sorting, searching, graph traversal, shortest paths, dynamic programming, and more.

Plus a **[Reference](reference/time-complexity.md)** section for time complexity and a quick cheat sheet.

## How to use it

Every page follows the same rhythm: what the thing is, how it works, a **Python implementation you can copy and run**, and its time complexity. Use the search bar (top right) to jump straight to a topic, and the moon/sun toggle to switch themes.

!!! tip "Everything here runs"
    All code samples are plain Python 3 and were tested before being written up. Paste them into a file or a REPL and they just work.
